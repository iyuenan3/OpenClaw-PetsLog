## 1. 测试环境配置

### 1.1 开发环境
```bash
# 环境要求
- Node.js >= 16.x
- npm >= 8.x
- 浏览器：Chrome 120+（推荐）

# 安装依赖
cd petslog-clean
npm install

# 启动开发服务器
npm run dev:h5

# 访问地址
http://localhost:5173
```

### 1.2 测试工具
```bash
# Playwright 测试
npm run test:e2e

# Playwright UI 模式
npm run test:e2e:ui

# 生成测试报告
npm run test:e2e:report
```

### 1.3 uniCloud 配置
- 空间：`openclaw-petslog-ns`
- 服务商：阿里云
- 云函数：需上传所有云函数

## 4. 自动化测试用例

详见：`tests/e2e/` 目录

### 4.1 测试文件结构
```
tests/
├── e2e/
│   ├── setup/
│   │   └── auth.setup.js      # 认证配置
│   ├── login.spec.js          # 登录测试
│   ├── pet-list.spec.js       # 宠物列表测试
│   ├── add-pet.spec.js        # 添加宠物测试
│   ├── medication.spec.js     # 用药记录测试
│   ├── weight-record.spec.js  # 体重记录测试
│   ├── reminder.spec.js       # 提醒功能测试
│   └── search.spec.js         # 搜索功能测试
```

### 4.2 运行测试
```bash
# 运行所有测试
npm run test:e2e

# UI 模式（可视化）
npm run test:e2e:ui

# 有头模式（看浏览器）
npm run test:e2e:headed

# 查看报告
npm run test:e2e:report
```
